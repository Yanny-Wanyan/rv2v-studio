# UI设计说明（评审版）

| 分类 | 原型 | 需求 |
|---|---|---|
| 顶部栏（Header） | 截图1：顶部栏（品牌 + 新建会话按钮） | - 展示产品名称与品牌标识<br>- 提供“新建会话”入口，一键重置当前会话状态<br>- 顶栏固定高度，视觉上与主内容区明确分层 |
| 视频窗-输入侧（Camera Feed） | 截图2：左侧视频窗（直播流/摄像头） | - 支持两种输入方式：直播流、摄像头<br>- 两个来源通过 segmented control 切换（圆角胶囊分段切换）<br>- 直播流支持输入地址后连接<br>- 摄像头支持拉起摄像头权限；拒权时展示错误态 |
| 视频窗-输出侧（Edited Output） | 截图3：右侧输出视频窗 | - 用于展示编辑后视频结果<br>- 与输入窗保持同尺寸、同样式头部与布局<br>- 无内容时显示占位提示 |
| 渲染状态标识（输出窗） | 截图15：右侧“实时渲染”状态变化（待机/渲染中/已完成/失败） | - 状态标识与渲染任务联动，不再静态写死<br>- 默认状态为`待机`（未发起任务）<br>- 发送指令后切换为`渲染中`（蓝色动效）<br>- 成功后切换为`已完成`（绿色）<br>- 失败后切换为`失败`（红色）<br>- 新建会话后重置为`待机` |
| 视频窗控制（暂停/退出） | 截图4：控制按钮可用态 + 禁用态 | - `暂停/继续`：仅摄像头模式可用<br>- `退出`：清空当前输入源并恢复初始空态<br>- 不同输入模式下按钮可用性自动切换（直播流禁用暂停） |
| 视频显示策略 | 截图5：竖屏视频在 16:9 窗口中的展示 | - 视频窗口固定为 16:9 容器<br>- 任意比例视频在窗口内完整显示（contain），不裁切主体<br>- 留白区域由背景承接，保证视觉稳定 |
| 直播/通话时长限制（1h） | 截图17：达到时长上限提示 + 截图18：对话已结束拦截 | - 直播流/通话启动后开始计时<br>- 超过 1 小时时提示：`已达到1小时最长交互时长，对话已结束，您可重新发起新对话`<br>- 达到上限后若继续发送，提示：`对话已结束`并拦截发送 |
| 通用异常提示（服务器繁忙） | 截图19：服务器繁忙提示弹窗 | - 预置统一异常提示能力<br>- 异常文案：`服务器繁忙，请稍后再试`<br>- 后续可与后端错误码（如 429/503）联动触发 |
| 指令输入框（Input Capsule） | 截图6：输入框区域（随机灵感 + 文本 + 发送） | - 支持文本输入编辑指令<br>- 回车可发送<br>- 发送按钮触发提交<br>- 随机灵感按钮可填充推荐文案 |
| 图片槽位（Image Slot） | 截图7：图片槽位空态 + 有图态 | - 支持上传参考图作为编辑条件<br>- 有图时显示缩略图与移除按钮<br>- 支持移除后回到空态 |
| 场景切换（Scene Tabs） | 截图8：场景按钮行 | - 提供五类场景：实时换装、风格演绎、外貌改造、质感变身、场景切换<br>- 点击场景打开对应面板<br>- 再次点击或点击外部区域可关闭面板 |
| 换装面板 | 截图9：换装选项标签 | - 文字标签选项：白T恤、皮夹克、灰色卫衣、汉服、学院风衬衫<br>- 点击标签后自动填充对应提示词到输入框 |
| 风格演绎面板 | 截图10：风格选项标签 | - 提供风格标签快捷选择：日系动漫、水墨国风、赛博朋克、油画质感、胶片复古、像素游戏<br>- 点击后将对应提示词写入输入框 |
| 外貌改造面板 | — | - 需人脸清晰<br>- 选项：变成光头、戴上墨镜、加耳环、变老30岁、变年轻到18岁、加络腮胡 |
| 质感变身面板 | — | - 把自己变成任何材质<br>- 选项：大理石雕塑、乐高小人、冰雕、赛博机械体、黏土动画、金箔覆盖 |
| 场景切换面板 | 截图11：场景选项标签 | - 替换背景<br>- 选项：咖啡厅、日式居酒屋、新闻演播室、城市天台、舞台聚光灯 |
| 历史记录（History） | 截图13：历史区 + 缩略图预览 | - 展示已发送指令历史<br>- 有参考图时显示缩略图<br>- 点击缩略图可查看预览弹层 |
| 会话重置（New Session） | 截图14：点击新建会话后的初始态 | - 清空视频输入与输出状态<br>- 清空输入框、图片槽位、历史记录、场景选中状态<br>- 恢复默认来源按钮状态 |
